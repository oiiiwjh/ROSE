# 会话总结与归档

你是 ROSE 系统的会话整理助手。请回顾本次会话的全部内容，生成两份结构化记录，并可选地提出 skill 优化建议。

## 输入

用户输入: $ARGUMENTS

可选参数：
- `--skip-optimize`: 跳过 skill 优化建议阶段
- 无参数: 执行完整流程（采集 → 归档 → 优化建议）

---

## Phase 1: 信息采集

并行执行以下操作收集本次会话的变更信息：

### 1.1 代码变更

运行以下命令获取变更概览：
- `git status`（查看未提交的变更）
- `git diff`（查看具体修改内容）
- `git log --oneline -10`（查看近期 commit）

### 1.2 知识产出

检查 `library/` 下的变更：
- `library/papers/` — 新增或修改的论文分析
- `library/topics/` — 新增或修改的方向综述
- `library/daily/` — 新增的每日推荐

### 1.3 系统变更

检查 `.claude/skills/` 和 `.claude/commands/` 下的变更：
- 新增/修改/删除的 skill 文件
- 脚本变更
- 配置变更（CLAUDE.md、config.md 等）

### 1.4 对话回顾

基于当前会话的对话上下文，总结：
- 讨论了哪些主题
- 解决了哪些问题
- 遇到了哪些错误及其解决方式
- 学到了哪些新概念或发现

---

## Phase 2: 生成两份记录

### 2.1 知识类记录（`library/daily/`）

**文件路径**: `library/daily/{YYYY-MM-DD}.md`

**定位**：只记录**科研知识产出**——论文分析、方向调研、学到的研究概念。系统/工程层面的内容（skill 优化、脚本修改、配置变更、bug 修复等）不属于此处，应归入 changelog。

**规则**：
- 如果文件已存在，**追加**新内容（不覆盖已有内容）
- 如果文件不存在，创建新文件并添加顶部标题 `# {YYYY-MM-DD} 每日记录`
- 如果本次会话没有知识产出（纯系统优化会话），可以跳过此文件

**追加内容格式**：

```markdown

## Session {HH:MM}

### 论文相关
- 分析了 [{论文标题}]({arxiv_url})（{arxiv_id}）
  - 核心发现：{一句话总结}
- Q&A: {关键问题} → {结论}

### 调研相关
- 调研了 {方向}，综述已保存至 library/topics/{slug}/

### 学习笔记
- {本次会话中学到的关键研究概念/学术发现}
```

注意：只包含实际发生的内容，没有的部分直接跳过对应章节。

### 2.2 系统功能类记录（`.claude/changelog/`）

**文件路径**: `.claude/changelog/{YYYY-MM-DD}.md`

**定位**：记录所有**系统/工程层面的变更**——skill 新增与修改、脚本变更、配置变更、bug 修复、流程优化等。这些内容不应出现在 `library/daily/` 中。

**规则**：
- 如果文件已存在，**追加**新内容
- 如果文件不存在，创建新文件并添加顶部标题 `# {YYYY-MM-DD} Changelog`
- 如果本次会话没有系统变更或 debug，可以跳过此文件的生成

**追加内容格式**：

```markdown

## Session {HH:MM}

### 变更记录
- [新增] {文件路径}: {描述}
- [修改] {文件路径}: {变更描述}
- [删除] {文件路径}: {原因}
- [修复] {问题描述} → {解决方式}

### Debug 记录
- **问题**: {错误描述}
- **原因**: {根因分析}
- **解决**: {修复方式}
- **影响文件**: {相关文件列表}
```

## Phase 3: Skill 优化建议（可选）

如果用户指定了 `--skip-optimize`，跳过此阶段。

### 3.1 分析优化点

基于本次会话中的 debug 经历、使用体验和发现的问题，检查：
- skill prompt 中是否有遗漏（如缺少错误处理、路径不正确）
- 脚本是否有 bug 或可改进的地方
- 流程是否有优化空间（如减少步骤、提高准确性）
- 是否需要新增 skill 或合并现有 skill

### 3.2 展示建议并让用户选择

如果有优化建议，使用 **AskUserQuestion（multiSelect: true）** 展示所有建议，让用户批量选择要采纳的项。

每条建议应包含：
- 优化目标（哪个 skill/文件）
- 当前问题
- 建议的改进

### 3.3 执行采纳的优化

仅对用户选中的建议执行修改：
1. 编辑对应的 skill/脚本文件
2. 将优化记录追加到 `.claude/changelog/{YYYY-MM-DD}.md`

### 3.4 如无优化建议

直接告知用户本次会话未发现需要优化的点。

---

## 输出摘要

完成后，向用户展示简要总结：
- 知识类记录保存路径及包含的条目数
- 系统变更记录保存路径及变更数量
- 优化建议执行情况（如有）
