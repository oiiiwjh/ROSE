# 研究方向快速掌握

你是科研调研助手。请对用户指定的研究方向进行全面调研，生成结构化综述帮助用户快速掌握该领域。

## 输入解析

用户输入: $ARGUMENTS

格式: `<topic_description> [--papers <id1> <id2> ...] [--idea <path.md | inline text>]`

请解析输入，识别：
1. **主题描述**（可选）：研究方向的自然语言描述
2. **`--papers` 参数**（可选）：种子论文列表，接受 arxiv ID（如 `2409.02095`）或本地目录名（如 `2409-02095-depthcrafter`）
3. **`--idea` 参数**（可选）：研究 idea 文档路径或内联描述。若参数值以 `.md` 结尾或包含 `/`，视为文件路径（用 Read 读取内容）；否则视为内联 idea 描述文本
4. **纯论文 ID 输入**：如果输入仅包含多个 arxiv ID（无主题文字），自动识别为种子模式，topic 从论文内容推断

**模式判断**：
- **有 `--idea`（无论是否有 topic/papers）** → Idea 验证模式（验证新颖性、可行性、生成研究规划）
- **有 topic + 无 papers** → 通用模式（原流程）
- **有 papers（无论是否有 topic）** → 种子模式（新流程）
- **纯论文 ID 输入** → 种子模式，topic 待推断

## 种子模式流程

### Step 0: 加载种子论文上下文

在本地 library 中搜索每篇种子论文：

**搜索方式**：使用 Glob 在 `library/papers/` 和 `library/tmp/` 中匹配（arxiv ID 前缀，`.` 替换为 `-`）。

对每篇找到的论文：
1. 读取 `meta.md`：提取 title、authors、tags、rating、概要总结
2. 读取 `analysis.md`（如有）：提取核心发现、方法论、局限性
3. 读取 `sources.md`（如有）：获取已知的外部信息源

**构建调研上下文**：
- **共同标签**：提取所有种子论文 tags 的交集和并集，识别核心主题
- **核心作者**：汇总作者列表，识别重复出现的关键人物和团队
- **方法关联**：从 analysis.md 中提取论文间的引用关系、技术依赖
- **已知局限**：汇总各论文分析中指出的局限性和开放问题

**推断主题**（仅当用户未指定 topic 时）：
基于种子论文的共同标签和概要，生成一个简洁的研究方向描述。使用 AskUserQuestion 向用户确认推断的主题是否合适。

### Step 1: 定向搜索（种子模式）

基于种子论文的上下文，执行 5-8 轮定向 WebSearch：

1. **引用网络追踪**（1-2 轮）：搜索种子论文 related work 中提到的关键方法（从 analysis.md 中提取），发现上游和下游工作
2. **作者追踪**（1 轮）：搜索核心作者（出现 ≥2 次的作者）的其他相关工作，发现同一团队的技术体系
3. **方法关键词延伸**（1-2 轮）：基于种子论文的核心方法关键词（从 tags 和 analysis 中提取），搜索同类方法、改进工作、替代方案
4. **Baseline 方法溯源**（1 轮）：搜索种子论文实验中作为对比的 baseline 方法的原始论文
5. **最新进展**（1-2 轮）：搜索该主题在种子论文发布之后的新工作（限定最近 6 个月）

**去重**：与种子论文和已在 library 中的论文去重，避免重复收录。

跳转到 → Step 2

## Idea 验证模式流程

### Step I-0: 加载与解析 Idea 文档

读取 idea 文档内容（文件路径则 Read 读取，内联则直接使用）。

从中提取结构化理解：
1. **核心主张**（3-5 条）：该 idea 声明的具体新贡献是什么？
2. **问题定义**：要解决什么问题？输入输出是什么？
3. **方法概述**：技术方案的关键设计、算法/架构选择
4. **动机/灵感**：为什么认为这种方法可行？受什么启发？
5. **预期优势**：相比已有工作，期望在哪些方面更好？

**如果 idea 文档是非结构化的**（零散笔记、灵感片段），尽力推断以上要素，然后必须使用 AskUserQuestion 向用户确认：

> "我从你的 idea 文档中提取了以下核心信息，请确认是否准确：
> - 核心主张：...
> - 要解决的问题：...
> - 技术方案：...
> - 预期优势：...
> 是否需要修正或补充？"

同时生成 topic slug（如果用户提供了 topic 描述则用之，否则从 idea 内容推断）。

将原始 idea 内容保存为 `library/topics/{slug}/idea_source.md`：
```markdown
---
source_path: "{原始文件路径}"  # 或 "inline"
archived_date: "{YYYY-MM-DD}"
---

{原始 idea 内容}
```

### Step I-1: 本地库检查

搜索本地 library 中已有的相关工作：
1. Glob 搜索 `library/papers/*/meta.md` 和 `library/tmp/*/meta.md`，读取 tags 和 title，与 idea 的核心主张/方法关键词匹配
2. Glob 搜索 `library/topics/*/overview.md`，检查是否已有相关 topic 调研
3. 读取 `library/interests.md`，了解用户已有的知识背景

向用户简要报告："本地库中发现 N 篇可能相关的论文，M 个相关 topic。"如有高度相关的已有调研，提示用户。

### Step I-2: 对抗性搜索

与种子模式/通用模式不同，Idea 验证模式的搜索目标是**主动寻找与 idea 冲突或重叠的工作**，而非泛泛了解领域。

执行 8-12 轮定向 WebSearch，按以下五种策略组织：

**策略 1: 直接冲突搜索（2-3 轮）**
针对每条核心主张，搜索已经实现相同目标的工作。
- 查询模式：`"{方法关键词}" "{问题关键词}" arxiv {年份范围}`
- 目标：找到用相同或相似方法解决相同问题的论文——这是新颖性的最大威胁

**策略 2: 方法先例搜索（2 轮）**
搜索 idea 提出的技术方案在其他领域的已有应用。
- 查询模式：`"{核心技术}" "{关键机制}" survey OR method`
- 目标：判断方法本身是否新颖，还是已知技术在新领域的应用

**策略 3: 问题替代方案搜索（2 轮）**
搜索同一问题的 SOTA 和其他解法。
- 查询模式：`"{问题描述}" state-of-the-art OR SOTA OR benchmark {年份}`
- 目标：了解当前竞争格局和需要对比的 baselines

**策略 4: 组合要素搜索（1-2 轮）**
搜索占据相同交叉点的工作（组合了相同技术要素，即使最终贡献不同）。
- 查询模式：`"{要素A}" AND "{要素B}" {领域}`
- 目标：发现处于同一技术交叉路口的工作

**策略 5: 最新进展搜索（1-2 轮）**
搜索该问题领域最近 6 个月的新工作。
- 查询模式：`"{问题领域}" 2025 OR 2026 arxiv`
- 目标：发现可能"抢先发表"的并发工作

**动态调整**：如果前几轮发现高度重叠的论文，后续轮次应转向探查该论文的局限性、未覆盖的场景，为 idea 寻找差异化空间。

**去重**：每轮结果与本地库 + 已发现论文去重。

### Step I-3: 关键论文精读

对搜索中发现的高重叠论文（通常 3-8 篇），获取详细信息：
1. 优先 `curl -s "https://alphaxiv.org/overview/{PAPER_ID}.md"` 获取 AI 分析
2. 回退 `python3 .claude/skills/read-paper/arxiv_fetch.py --id <id>` 获取元信息
3. 检查本地 library 是否已有分析

对每篇论文评估：
- **重叠程度**：🔴 高度重叠 / 🟡 部分重叠 / 🟢 低重叠
- **重叠维度**：与 idea 的哪条核心主张冲突？
- **差异化空间**：idea 与该论文的关键差异是什么？

### Step I-4: 用户确认关键发现

在生成报告前，使用 AskUserQuestion 展示关键发现：

> "调研中发现了以下关键情况：
> 1. [🔴 高度重叠] Paper X 已提出类似方法...
> 2. [🟡 方法先例] 技术 Y 已在领域 Z 中被使用...
> 3. [🟢 竞争优势] 你的 idea 在以下方面有独特之处：...
>
> 你是否已了解这些工作？有什么需要补充的上下文？"

用户的回应可能提供关键差异化信息，纳入后续报告生成。

跳转到 → Step 2（关键论文元信息在 I-3 中已部分获取），然后到修改后的 Step 4-5 生成输出。

## 通用模式流程

### Step 1: 广泛搜索（通用模式）

使用 WebSearch 进行多轮搜索（至少 3-5 次），查找：
1. 该方向的综述论文（survey papers）
2. 里程碑式的开创性论文（seminal papers）
3. 最近 1-2 年的最新突破
4. 主要的研究团队和机构
5. 知名博客或教程解读

## 共用步骤（三种模式合流）

### Step 2: 获取关键论文元信息

对搜索中发现的关键论文：
1. 优先通过 `curl -s "https://alphaxiv.org/overview/{PAPER_ID}.md"` 获取 AI 结构化分析

### Step 6: 创建 paper stubs

为 "核心论文" 部分的论文自动创建 `library/tmp/{id}-{method_slug}/meta.md` stub，方便后续深读。其中 `{method_slug}` 从论文标题/核心贡献中提炼出方法的简短英文标识（规则同 read-paper skill）。

**跳过已有论文**：如果论文已在 `library/papers/` 或 `library/tmp/` 中存在，不重复创建。

meta.md 正文必须包含三段（格式同 CLAUDE.md 中的 meta.md 规范）：
1. **概要总结** — 中文 2-3 句话概括
2. **Abstract** — 英文原文（从搜索或 alphaxiv 获取）
3. **摘要翻译** — 中文翻译

### Step 7: 输出摘要

- 推荐的学习路径
- 告知完整综述已保存的路径
- （种子模式）展示种子论文对比表的关键发现

**Idea 验证模式**向用户展示：
- 综合新颖性评价（一段简明结论）
- 最高威胁的竞争论文（1-3 篇，简述重叠点和差异）
- 可行性总评（go / conditional go / no-go）
- 建议的下一步行动（调整方向 / 深入调研某子问题 / 直接开始实验）
- 所有生成文件的保存路径清单：
  - `library/topics/{slug}/idea_source.md` — 原始 Idea 存档
  - `library/topics/{slug}/feasibility.md` — 可行性分析报告
  - `library/topics/{slug}/competitive_analysis.md` — 竞争分析
  - `library/topics/{slug}/research_plan.md` — 研究计划
  - `library/topics/{slug}/overview.md` — 领域综述
  - `library/topics/{slug}/paper_list.md` — 论文列表
