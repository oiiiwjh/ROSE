# v1.0.5 变更日志 — 2026-04-21

## Blog 支持与内容类型扩展

### 新功能
- `/read-paper` 扩展支持 URL 输入，自动识别 blog/project-page/thread/newsletter 类型
- Blog 分支：WebFetch 抓取页面内容、元信息提取、关联论文检测
- `library/blogs/` 收藏目录：blog 收藏至此，论文仍收至 papers/
- Blog 分析模板适配：「实验设计与结果分析」→「实证与演示」+ 新增「关键 Insight」段
- `library/interests.md` 新增 `## Followed Blogs` 固定博客源配置
- `/daily-papers` 增加博客源检查与推荐（标注 `[Blog: {站点名}]`）

### 数据格式变更
- meta.md frontmatter 新增字段：`content_type`（默认 paper）、`source_site`（blog 类型）、`related_papers`（blog 类型）
- 博客目录命名：`{YYYYMMDD}-{slug}`（无 arxiv ID 时）
- `generate_index.py` 扫描范围增加 `library/blogs/`

### 阅读状态与阅读列表增强
- Library 索引（papers/blogs）新增「阅读状态」列：待阅读(⬚) / 已分析(📝) / 计划阅读(📖) / 已精读(✅)
- `/read-paper` 分析完成后（Step 8c）询问是否加入阅读列表（可选优先级）
- `generate_index.py` 新增 `parse_reading_list()` 和 `reading_status()` 函数
- CLAUDE.md 新增「阅读状态定义」表

### Bug 修复
- `read-paper` Step 0 搜索范围补充 `library/blogs/`（之前遗漏）
- `read-paper` Step 0b 完整性检查范围扩展到 blogs/
- `project-page` URL 自动判断规则从子串匹配收紧为域名模式匹配

### 向后兼容
- 已有论文的 meta.md 无需修改，`content_type` 缺省视为 `paper`
- 所有现有 skill 的论文处理逻辑不受影响
